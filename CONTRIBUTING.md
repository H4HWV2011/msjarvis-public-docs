## How to Contribute

### Documentation Improvements

We welcome contributions to:
- Architecture explanations
- API documentation clarifications
- Tutorial additions
- Error corrections
- Translation to other languages

### Process

1. **Fork this repository**
2. **Create a branch**: `git checkout -b docs/your-feature`
3. **Make your changes**
4. **Test**: Ensure markdown renders correctly
5. **Commit**: `git commit -m "docs: describe your change"`
6. **Push**: `git push origin docs/your-feature`
7. **Open Pull Request** with clear description

### Pull Request Guidelines

- Keep changes focused on documentation
- Use clear, concise language
- Follow existing markdown formatting
- Add examples where helpful
- Update table of contents if adding sections
